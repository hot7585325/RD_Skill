Azure的計費機制

在 Azure OpenAI Service 中，token 是依照輸入與輸出的文字量計算，不同模型與媒體類型（文字、圖片、語音）計費方式不一：

文字模型（如 GPT-4、GPT-4-turbo、GPT-3.5

  • 分為 Prompt tokens（輸入） 與 Completion tokens（輸出）。
  • 費用依據模型而異（例如 GPT-4-turbo 約 $0.01/1K tokens 輸入，$0.03/1K tokens 輸出）。

圖片生成（DALL·E）

  • 按「生成張數」與「解析度」計費，不以 token 計算
  • 256×256、512×512、1024×1024 各有不同費率（約 $0.016~$0.08 / 張）

語音 / 語音轉文字（Speech to Text / TTS）

  • 依「秒數」或「文字字數」計價，不計 token
  • Speech to Text：約 $1 / 小時音訊。
  • Text to Speech：約 $16 / 百萬字元（視語音品質不同）

強制指定語言以避免混用

方式一：Prompt 直接明確指定

「請全程以繁體中文回答，請勿出現任何簡體字。」或指定 #zh-tw

方式二：API 參數語言標籤

  • - Azure TTS/ASR、翻譯 API：locale（如 zh-TW、zh-CN）
  • - Azure Speech Service：在 session 設定強制指定 locale: zh-TW
  • - Azure Translator：REST API 輸出時設 to=zh-Hant（繁體中文），或語言自設定

實作 RAG

RAG 技術架構與 Azure 實作配置

RAG（Retrieval-Augmented Generation, 擷取增強生成）是現代 AI 中文件與知識庫對話的主流架構。透過檢索引擎（如 Azure AI Search）先將外部特定文件分塊、索引、向量化，再將相關段落作為 prompt input 提供給 LLM，在回答時僅考慮限定資料源，保證安全性、專業性與權限隔離

Azure RAG 標準實作過程

  1. 文件上傳：支援 txt、md、html、docx、pptx、pdf 等主流格式，上傳至 Azure AI Foundry/AI Search 或 Blob Storage。
  2. 分塊/嵌入：採用 phrase-based（語意型）或 fixed-size（按 token 數）分段，Azure 支援 Markdown 教材自動區塊劃分。
  3. 向量化索引：以內嵌（Embedding）模型（推薦 text-embedding-ada-002、gpt-v 等）將內容轉向量儲存於 Azure Search 索引。
  4. 查詢限制：查詢/問答僅限制於該資料索引，不會超出特定文件集（支援 REST API 篩選/分組）。
  5. 權限篩選：搭配 Azure AD/Entra 制定細緻權限（如文件層級存取），每次查詢都經存取條件過濾。
  6. Unity/前端整合：API 回應可精確揭露來源段落、頁碼、引用原文、甚至附圖及原始檔下載連結。
  7. 實作可使用 LangChain/LlamaIndex/Semantic Kernel 開源方案串接

RAG 文件範圍控制與參數

  • 可上傳資料
  • 可設搜尋範圍（資料夾、Blob Container、文件屬性欄位）
  • 支援向量搜尋、語意搜尋、混合搜尋
  • 支援文件層級存取權（Entra 群組）
  • 查詢 API 支援欄位濾選（filter 參數
